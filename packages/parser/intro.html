Parser 套件

Parser 套件扩展核心视图类 能够解析使用任何类型的模版.

简介

Parser 改变不了你使用视图模版的方式, 但他允许使用任何PHP的模版. 这个套件附带了解析驱动，如 Dwoo, Haml, Jade, Lex, Markdown, Mustache, PHPTal, Smarty and Twig.

解析器的扩展名是由使用的视图决定的. 标准看法,用PHP解析的扩展名应该是 .php. 这是默认的, 如果没有指定扩展名, 会假定 .php 为扩展名.

其他的一些独有的扩展名, 如何 foo.mustache 会使用 Mustache 驱动, foo.tpl 使用 Dwoo 驱动, foo.lex 使用 Lex 驱动, 等等. 在配置中可以修改默认扩展名).

安装

Parser 套件包含在 Fuel 下载包中. 第一次使用时必须增加一下配置:

'always_load' => array(
	'packages' => array(
		'parser',
	),
),

想要支持解析, 需要通过 composer来安装. 仅仅增加库文件到 composer.json 里然后运行php composer.phar 安装:

{
    "require": {
        "dwoo/dwoo" : "*",
        "mustache/mustache" : "*",
        "smarty/smarty" : "*",
        "twig/twig" : "*",
        "mthaml/mthaml": "*",
		"pyrocms/lex": "*"
    }
}

注意: Markdown 解析是默认安装的, FuelPHP 核心类 Markdown也会使用.

如果库文件不能通过 composer安装在 APPPATH/vendor/lib_name (使用 lib_name), 那么你必须自己下载. 不要做任何修改, 在 vendor/lib_name 目录保持原装 这样容易更新 (由于部分会自动加载).

配置

在config/parser.php 文件中配置解析器，需要一下项:

参数 类型 默认 详细描述
extensions array (扩展数组和解析视图类) 定义有效视图驱动扩展.
如: 'md' => 'View_Markdown'

对于每一个视图驱动程序，配置文件包含一个与驱动程序相同名称的数组条目，其中列出了驱动程序特定的配置 (限于所支持的配置项驱动。这可能是所有配置解析器支持的一个子集).

运行环境配置

所有驱动有个 parser() 方法 能让你访问Parser 对象. 这是个惯例而非必须的，且不同的第三方驱动可能有所不同.

// Clear the cache for a specific Smarty template
$view = View::forge('example.smarty');
$view->parser()->clearCache('example.smarty');

// Example static usage
View_Smarty::parser()->clearCache('example.smarty');

这也可以使用自定义配置传递给驱动.
